Estrategia GitFlow - DevSecops

¿Qué es GitFlow?

GitFlow es un modelo de flujo de trabajo para Git que define un conjunto estricto de reglas de ramificación diseñadas para la gestión de versiones de software. Utiliza ramas específicas para diferentes propósitos, lo que facilita el desarrollo paralelo, la preparación de versiones y el mantenimiento de correcciones rápidas en producción.

Ramas Principales

  • master: Rama principal de producción, siempre estable.
  • develop: Rama principal de desarrollo, donde se integran las nuevas funcionalidades.

Ramas de Soporte

  • feature: Ramas para el desarrollo de nuevas funcionalidades.
  • release: Ramas para la preparación de nuevas versiones.
  • hotfix: Ramas para la corrección de errores en producción.

Flujo de Trabajo

  1. Se crea una rama feature a partir de develop.
  2. Se desarrollan las funcionalidades en la rama feature.
  3. Se fusiona la rama feature con develop.
  4. Se crea una rama release a partir de develop.
  5. Se realizan las últimas correcciones y pruebas en la rama release.
  6. Se fusiona la rama release con master y develop.
  7. Se crea una etiqueta (tag) en master para la versión.
  8. Se crea una rama hotfix a partir de master.
  9. Se corrige el error en la rama hotfix.
  10. Se fusiona la rama hotfix con master y develop.
  11. Se crea una etiqueta (tag) en master para la corrección.
